System

Flats retail cost

Edit the Retail value for each weight tier (oz). This updates flat_rate_costs.rate_retail. Presort reject charge (per piece) is used on the postage invoice for WS3 reject counts (distributed across 1–13 oz and cost centers by flat volume; included in totals).

Priority Mail parcel retail (zones)

Upload the compact USPS-style zone × weight workbook (zones across the top, pounds down the side). Choose the effective date for this tariff (inclusive): billing is priced using the latest upload on or before each piece date until you publish a newer one.

Import customers (Raw Export)

WS3 Raw Export workbook: column A = Customer, B = Code, N = Parent Company. Parent account numbers are resolved by matching Parent Company text to column A (same as a VLOOKUP to Code). Replaces all rows in customers.

WS3 profile → parent account

Map each NetSort profile to a rollup account (parent company or main account) and set an optional reject fee (e.g. per reject). Parent accounts have child accounts; main accounts have none. Import WS3 Customer Mail Detail files via input/ first; then configure here.

(large customer lists load on demand.)

Unmatched accounts → create/assign

When imports contain a customer number that is not in customers, it shows as Unmatched. Use this table to give the number a name and optionally associate it to a parent account.

(avoid scanning huge tables on each visit.)
