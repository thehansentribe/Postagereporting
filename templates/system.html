System

Flats retail cost

Edit the Retail value for each weight tier (oz). This updates flat_rate_costs.rate_retail. Presort reject charge (per piece) is used on the postage invoice for WS3 reject counts (distributed across 1–13 oz and cost centers by flat volume; included in totals).

Pricing terms (supplier → EFD → customer)

Discounts and fees used by profit reports and invoices. Each report uses the revision in effect on the report end date; values typed into the dashboard toolbar override these for a single report. Save with a new effective date to record a terms change without losing history.

Notice 123 rate case

Upload a USPS Notice 123 price-change .zip. Files are extracted to Notice123/ and the system imports Priority Mail retail, First-Class flats retail, and commercial presort rates. Choose the effective date for this tariff (may be in the future): billing uses the latest upload on or before each piece date.

Current flats retail & presort rates

Read-only view of flat mail rates active as of today. Click to load the latest tariff on or before the current date.

Current parcel (Priority Mail) zone rates

Read-only view of Priority Mail retail zone × weight rates active as of today, plus flat-rate envelopes and boxes.

Import customers (Raw Export)

WS3 Raw Export workbook: column A = Customer, B = Code, N = Parent Company. Parent account numbers are resolved by matching Parent Company text to column A (same as a VLOOKUP to Code). Replaces all rows in customers.

WS3 profile → parent account

Map each NetSort profile to a rollup account (parent company or main account) and set an optional reject fee (e.g. per reject). Parent accounts have child accounts; main accounts have none. Import WS3 Customer Mail Detail files via input/ first; then configure here.

(large customer lists load on demand.)

Unmatched accounts → create/assign

When imports contain a customer number that is not in customers, it shows as Unmatched. Use this table to give the number a name and optionally associate it to a parent account.

(avoid scanning huge tables on each visit.)

Backup & Restore

Download backup saves a .zip of the database and report data. Restore loads such a zip back in — useful for moving to another computer or recovering data. Restoring stages the data and takes effect after the app (or Windows service) is restarted; the current database is snapshotted to postage.db.bak-restore-* first, so a restore can be undone.
